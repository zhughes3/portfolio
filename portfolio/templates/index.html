Compiler Design

My most enjoyable class this semester has been COMP 520, Compilers, with Dr. Jan Prins. This is known as being one of the most difficult courses in the Computer Science curriculum here at UNC. And difficult it is - like practically every other 400+ level class I've taken. But this one has been the most interesting because it does an amazing job of combining advanced theory with practical software engineering experience. At the end of the semester, each of us in the class will have built a compiler for a subset of the Java Programming Language, which we call "miniJava".

miniJava supports classes, int and boolean types, along with many other aspects of Java. It does not support inheritance of method overloaded in its current form but can be updated in the future. This assignment was broken down into four major components of the compilation process:

  1. Syntactic analysis
  2. Abstract Syntax Trees
  3. Contextual and Type Analysis
  4. Code Generation

During each step, we created the necessary classes needed. Over the next few weeks, I will go into greater detail about the steps I took to build this compiler, hopefully enlightening many of you readers in the process.
